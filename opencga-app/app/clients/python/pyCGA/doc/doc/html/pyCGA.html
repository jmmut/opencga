PyCGA¶

If you are using PyCGA you have to know the first you need is authenticate yourself in the system, so you will be recognize by the application and you will be granted to perform the queries you are allowed to. Once you are finished to work with the application you should log out. Both actions can be done using a simple command line application: PyCGALogin.

PyCGALogin¶

This tool provide the interface to log in and logout to the system

usage: pyCGALogin [-h] {login,logout} ...
Sub-commands:
login

Use this command to log in the system and get a Session Id

usage: pyCGALogin login [-h] --host OpenCGA Host [--instance instance] --user
                        User --pwd Password
Options:
--host Optional
--instance=opencga
 Optional
--user Required. User name
--pwd Required. User password
logout

Use this command to log out, using your current Session Id

usage: pyCGALogin logout [-h] --user User
Options:
--user Required. User name

If you have already your session Id, you can start to query the Catalog. The aim of pyCGA is provide very simple command lines to perform complex actions in Catalog. In this guide you are going to find how to call these programs from the linux terminal and several example to use cases. PyCGA application it is compounded of 12 tools distributed in 5 sections:

  • User Tools
  • File Tools
  • Sample Tools
  • Individuals Tools
  • Variable Sets Tools

User Tools¶

usage: pyCGA users [-h] {create,delete,acl} ...
Sub-commands:
create

Use this command to create a new user, only administrators are allowed to perform this action

usage: pyCGA users create [-h] --user user --name name --email email --pwd
                          password --org organization
Options:
--user Required. User name
--name Required. User full name
--email Required. Uses email
--pwd Required. password
--org Required. Organization
delete

Use this command to delete a existing user, only administrators are allowed to perform this action

usage: pyCGA users delete [-h] --userId UserId
Options:
--userId Required. User name
acl

Use this command to check the permission of an existing user

usage: pyCGA users acl [-h] --userId UserId
Options:
--userId Required. User name

File Tools¶

methods for files

usage: pyCGA files [-h]
                   {index,info,query_stats,search_by_sample,link_file} ...
Sub-commands:
index

index a bam or vcf file

usage: pyCGA files index [-h] --fileId file id [--outdir output directory]
                         [--annotate] --studyId StudyId
Options:
--fileId Required
--outdir Required
--annotate=False
 Variants will be annotated
--studyId Required
info

reetrieve info on a file given its id

usage: pyCGA files info [-h] --fileId file id --subset file id --field file id
Options:
--fileId Required
--subset Required
--field Required
query_stats

retrieve info on a file given its id

usage: pyCGA files query_stats [-h] --studyId study id --fileType file type
                               ALIGNMENT o VARIANTS --query query e.g
                               BAM_HEADER_MACHINE.MACHINE
Options:
--studyId Required
--fileType Required
--query Required
search_by_sample

retrieve the ids and the path of the files

usage: pyCGA files search_by_sample [-h] --studyId study id --fileType file
                                    type ALIGNMENT or VARIANTS --sampleName
                                    [Sample Name [Sample Name ...]]
                                    [--path path]
Options:
--studyId Required
--fileType Required
--sampleName Required
--path=~^ Optional. Path under the search will be done. i,e: ~^by_date/*
link_file

link a single file

usage: pyCGA files link_file [-h] --studyId study id --uri full path to file
                             [--sampleId catalog sample id for file] --path
                             Path in catalog
Options:
--studyId Required
--uri Required
--sampleId Not Required
--path Required

Variable Sets Tools¶

usage: pyCGA variables [-h] {create} ...
Sub-commands:
create

Create new annotation Set

usage: pyCGA variables create [-h] --name NameOfVariableSet --studyId studyID
                              --json json [--format format]
Options:
--name Required. Name of the new variable set
--studyId Required. Study Id were the variable sets will be stored
--json Required. Json file with the variableSet
--format=OPENCGA
 

Default: OPENCGA

Possible choices: AVRO, OPENCGA
